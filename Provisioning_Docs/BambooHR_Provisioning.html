Configuring Provisioning for BambooHR

This guide provides the steps required to configure Provisioning for BambooHR.

Notes

  • This integration cannot create new users in BambooHR. Assigning an Okta user to BambooHR with Provisioning enabled will result in an error. Instead, import existing users from BambooHR first to avoid this error.

  • If you want to republish the Mapping Template via SU, turn off Push User Attributes feature to avoid provisioning errors, or you can retry failed tasks after the template is republished.

Configuring BambooHR as a New App

  1. Obtain a BambooHR API Key. To do this:

    • Login to BambooHR as an administrator.

    • Select Account Settings in the upper right corner, then select API Keys.

    • Select Add a new key, or uses an existing one if you have one is the list ov Valid API Keys

    • Make a copy of the Key to enter in Okta later in this procedure.

  2. In Okta, from the Admin dashboard, select Applications > Add Application.

  3. Search for the BambooHR app, then click Add.

  4. Under BambooHR’s General Settings:

    • Applications Label: Add an application label.

    • Subdomain: Enter your BambooHR subdomain. For example, enter company for http://company.bamboohr.com/.

    • Click Next.

  5. Configure your Sign-On Options on the next tab, then click Next.

  6. Configure your Provisioning Settings as follows:

    • Click Enable provisioning features.

    • API Key: Enter the key you copied from BambooHR (step 1) into this field.

    • Parameter to Use for Groups: You can select any of the following parameters:

      • Departments

      • Locations

      • Divisions

    • Pre-Start Interval: Enter the interval in days, so that users with Hire Date ahead of current date and within this interval could be considered active and imported to Okta. For example, if you enter 7, the users with a Hire Date later than one week ahead of current date will not be imported.

  7. Scroll down and select the Provisioning Features you want to enable, either Profile Master or Push User Attributes, then click Next. Note that you cannot enable both of these at the same time, select only one.

  8. You can now assign people to the app (if needed) and finish the application setup.

  9. In Okta, from the Admin dashboard, select Directory > Profile Editor.

  10. Select the APPS section in the left navigation bar, then find your app in the list.

  11. Check the list of attributes, and if you decide you need more, click Add Attribute. A list of extended attributes will appear:

  12. Select the attributes you want to add, then click Save.

  13. You can now import and push these user attribute values to/from BambooHR
