Configuring Provisioning for Samanage

This guide provides the steps required to configure Provisioning for Samanage.

Overview

Currently with Workday application, we do not allow Okta to update Workday Attributes. With this new feature we allow Okta, Active Directory, or any other Profile Master to update Phone Number and Email Workday attributes. No other Workday attributes are supported at this time.

To enable Workday write back for email and phone number. Please enable the following feature flags.

  • PROV_WORKDAY_PROFILE_UPDATE
  • ALLOW_BOTH_PROFILE_MASTERING_AND_PUSH
  • ATTRIBUTE_LEVEL_MASTERING

Prerequisites

Set up a Workday tenant, as described in the Workday Deployment Guide.

Okta Setup

  1. In Okta, open the Workday app, and navigate to the Provisioning tab:

  2. Scroll down and select Enable for Update User Attributes:

  3. Under Profile Attribute and Mappings click Edit Attributes:

  4. We need to now add workday attributes that are currently supported for write back. This is done by discovering attributes from workday and mapping to relevant Okta attributes. Click Add Attribute:

  5. Click Refresh Attributes:

  6. Add the required Workday attributes, then click Save:

    Note that each attribute has three different data types which are explained below. In this table, we are using the attribute Work Email as an example:

    Attribute Name Type Description Required/Optional Comments Sample Values
    Work Email String workEmail Required Map to Okta user attribute OktaCustomWorkEmail
    Set Work Email As Primary boolean setWorkEmailAsPrimary Optional If Work email that is pushed needs to be set as Primary, Set value as true. If not set, Default is false true
    Set Work Email As Public boolean setWorkEmailAsPublic Optional If Work email that is pushed needs to be set as Public, Set value as true. If not set, Default is false true

    Workday Contact information has 2 parameters: Primary and Public.

    Primary: This parameter defines if the contact information is of type Primary or Additional.

    Public: This parameter defines if the contact information visibility is Public or Private.

    Note that Workday requires a Primary value for any attribute. So if Okta pushes any attribute as NOT primary and if the Workday Worker does not already have Primary set in Workday, then Okta will set the current value as Primary.

    A table at the end of this article lists the Workday attributes that are currently supported.

  7. Map the newly added attribute(s) to relevant Okta attribute(s). Click Map Attributes

  8. Make sure you select Okta to Workday, then map attributes:

    Note that the Okta attributes that are mapped to Workday needs attribute level mapping with either Okta or any other Profile master such as Active Directory. See the example below:

    Under Master Priority we can set any available Profile master to be the source of truth. In this example we will set it to Active directory since we are pushing from AD to OKTA to Workday.

This completes the setup for Workday Write Back. Now we can successfully push email and phone attributes to Workday.

Known Issues

  1. Unlike the Workday UI, the Workday API does not have format validation/rules for phone number or email (Ex: Phone number format must be 3 Digits + Optional dash (or dot) + 4 digits).
