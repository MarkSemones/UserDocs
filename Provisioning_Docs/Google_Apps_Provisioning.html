Configuring Google License Management

This document currently only describes how to configure Google License Management. We plan to update this document to include more provisioning configuration information in the near future.

Overview

When a user is assigned to Google Apps you can manage all licenses and deprovisioning actions, centrally and granularly, within Okta. When a user is given a Google Apps license in Okta (individually or within a group), an Okta user instance is auto-created in the Google admin console. Any updates created in Okta are subsequently pushed to Google.

Requirements

Before you use this option, you must do the following:

  • Un-check the Auto-assign the following license to all currently unassigned users and users subsequently created button in your Google admin console account.

  • Have Google Apps provisioning enabled in Okta.

Configuring Google License Management

  1. In Okta, select the People tab for the Google Apps app.

  2. Click the right-side Edit button. The Edit User Assignment page appears.

  3. Select true from the Manage licenses on create update drop-down menu.

  4. Click the appropriate check-boxes to assign the license(s).

Caution: Only certain groups of Google licenses can be individually selected, as some groups provide mutually exclusive options. For example: You can assign only one license selection at a time for Google Apps and Google Drive. Choosing more than one results in an error on the Tasks page, and only the first selection is honored.

Deactivating a Google Apps User

When you need to deprovision or deactivate an Okta user from Google Apps, you can do so without compromising the user's Google account. Control this element under Deactivation options.

Deactivation options are:

  • If neither of the check boxes are checked, the user is suspended in Okta and in their corresponding Google account.

  • Do not suspend user: Checking this box prevents the user's account from being suspended in Google after deprovisioning in Okta. The user's Google account remains active.

  • Remove all Google Apps licenses: Checking this box insures that, when deprovisioned, Google licenses for this user no longer exist in Okta.

Troubleshooting Tips

Chiclets for apps for which users do not have a license may appear on their Okta Home page. Clicking these chiclets results in an error message.
