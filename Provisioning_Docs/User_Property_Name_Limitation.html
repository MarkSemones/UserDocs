User Property Name Limitations

This document describes how Okta deals with non-alphanumeric characters in Schema Discover and Services.

Okta allows only alphanumeric characters and underscores in its User Property names, additionally, they cannot start with a digit.

This constraint creates a limitation for Schema Discovery and Services, which allows non-alphanumeric characters in their user attributes.

For cases when Service does support non-alphanumeric characters, Okta converts these characters to their unicode number, then adds a ”_u” at the beginning.

Here are some examples:

Passport# -> Passport_u0023

custom$ -> custom_u0024

For properties that start with a digit, a preceding "p" character is added:

1stParam -> p1stParam

The provisioning for such properties still works.

The Display Name will not be modified.

Note: You must use a modified property name when setting up Mapping.
