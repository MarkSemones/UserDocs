How to Configure SAML 2.0 for Deputy

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. Login to your Deputy account as an administrator.

  2. Go to https://[yourSubDomain].deputy.com/exec/config/system_config.

  3. Navigate to Security Settings, then click Edit:

  4. The Security Settings page appears. See screen shot at end of instructions for reference.

  5. For Enable Social Login, select ON.

  6. OpenSSL Certificate: Copy and paste the following certificate (PEM text format):

    Sign into the Okta Admin Dashboard to generate this variable.

  7. Deputy Token to create user on the fly. This is optional and is required for Just In Time (JIT) provisioning.

    To get the Access Token:

    1. Click the here link, or go to https://[yourSubDomain].deputy.com/exec/devapp/oauth_clients:

    2. Click New OAuth Client:

    3. Enter the following:

      • Name: Enter Okta.

      • Redirect Uri: Enter okta.com.

      • Click Save This OAuth Client.

    4. Click Get An Access Token:

    5. Copy your Access Token from the popup message:

    6. Enter your Access Token on the Security Settings page.

  8. Click Save Settings:

  9. Done!

    Notes:

    IdP-initiated flows and Just in Time (JIT) provisioning are supported.

    SP-initiated flows are not supported.
