How to Configure SAML 2.0 for StatusPage.io

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. Log into your StatusPage.io account.

  2. In the top right corner, first select your email, select Manage Account from the dropdown menu, then select the Single Sign-on tab:

  3. At the top of the Enable SSO for Viewing The Status Page form, make a copy of the values for ACS URL/Consumer URL and the EntityID/Audience URI.

    Alternatively you can click the service provider metadata XML file for this page link to see the raw SAML metadata and get the values from there, as shown below:

  4. In the Enable SSO for Viewing The Status Page form, enter the following information:

    • SSO Target URL: Copy and paste the following (see screen shot at end of step for reference):

      Sign in to the Okta Admin app to have this variable generated for you.

    • Certificate: Copy and paste the following (be sure to include the BEGIN header and END footer):

      Sign into the Okta Admin Dashboard to generate this variable.

    • Click SAVE CONFIGURATION.

  5. In Okta, select the General tab for the StatusPage app:

    • Enter the StatusPage.io’s ACS Url and entityID you made note of in step 3.

    • Click Save.

  6. Done!

    Notes:

    IdP-initiated flows and SP-initiated flows are supported.

    Just In Time (JIT) Provisioning is not supported.
