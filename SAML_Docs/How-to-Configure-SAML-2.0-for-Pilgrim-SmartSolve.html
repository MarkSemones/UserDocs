How to Configure SAML 2.0 for Pilgrim SmartSolve

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. Send an email to the Pilgrim Quality Software support team to request SAML 2.0  access. 

  2. Copy the following IDP Metadata into a text file. Save it with the file name metadata.xml. Attach this file to the email message.

    Sign in to Okta Admin app to have this variable generated for you.

  3. Download and attach the following x.509 Certificate to the email:

    Sign in to the Okta Admin dashboard to generate this value.
  4. You can configure this value. The general format is https://yourDomain.com/identityProvider. A sample SmartSolveBaseURL is https://smartsolve.pilgrimasp.com/OktaSAML.

  5. Include your Assertion Consumer Service URL in the format shown below.

    [SmartSolveBaseURL]/AssertionConsumerService.ashx

  6. Include your Login Service URL in the format shown below.

    [SmartSolveBaseURL]/LoginService.ashx

  7. Include your Logout Service URL in the format shown below.

    [SmartSolveBaseURL]/LogoutService.ashx

  8. Include your External Key in the format shown below.

    urn:smartsolve:[SmartSolveCustomerName]ServiceProvider

  9.  Pilgrim Quality Software will process your request. After receiving a confirmation, you can start assigning people to the application.
  10. Done!

Note: Pilgrim Quality Software supports IdP-initiated and SP-initiated flows. Just In Time (JIT) provisioning are not supported.
