How to Configure SAML 2.0 for iRise

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

Before you begin

Read this before you enable SAML

Currently "Logout" is not supported for the iRise application (an iRise issue). If you use "Logout" you will be redirected to the iRise Dashboard.

  1. Send an email to the iRise support team (support@irise.com) and request that they enable SAML 2.0 for your account.

  2. Provide them with the following information:

    • IDP Issuer/Entity ID: Copy and paste the following:

      Sign into the Okta Admin Dashboard to generate this variable.

    • Login URL: Copy and paste the following:

      Sign into the Okta Admin Dashboard to generate this variable.

    • Certificate: Copy and paste the following:

      Sign into the Okta Admin Dashboard to generate this variable.

  3. The iRise support team will process your request. After receiving a confirmation email, you can start assigning people to the application.

  4. Done!

    Notes:

    SP-initiated flows and IdP-initiated flows are supported.

    Just In Time (JIT) provisioning is not supported.

    For SP-initiated flows, open your Base URL: https://<YourSubDomain>.dc.irise.com
