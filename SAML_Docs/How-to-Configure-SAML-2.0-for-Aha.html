How to Configure SAML 2.0 for Aha!

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. Generate the following IDP Metadata and save to a file named metadata.xml:

     
  2. Log into Aha! as an Administrator.
  3. Navigate to Account Settings > Security and Single Sign-On > Single Sign-On.
  4. Select SAML 2.0.

  5. Enter the following values (see screen shot at the end of the step for reference):

      • Name: OKTA
      • Select the Metadata File checkbox.
      • Click Browse and navigate to the metadata.xml you created in step 1.

  6. Done!

    Notes:

    IDP-initiated flows, SP-initiated flows, and Just In Time (JIT) provisioning are all supported.

    For SP-initiated flows, go to https://<yourSubDomain>.aha.io/session/new and select Log in with Okta. Where <yourSubDomain> is your Aha! subdomain.
