How to Configure SAML 2.0 for New Relic

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

Before you begin

Read this before you enable SAML

Enabling SAML will affect all users who use this application, which means that users will not be able to sign-in through their regular log-in page. They will only be able to access the app through the Okta service.

Backup URL

New Relic does not provide backup log-in url where users can sign-in using their normal username and password. You can call New Relic support to turn off SAML, if necessary.

  1. Sign in to New Relic as a user with rights to edit SAML configurations.

  2. Go to Account Settings.

  3. Next, go to Security and authentication > Sign sign on > Configure.

  6. In New Relic, copy the Assertion Consumer URL and paste it into the Login URL field in Okta Configuration, under the General tab.

  7. Save the configuration in both Okta and New Relic.
  8. Click Test SAML Login in New Relic to test your configuration.
  9. After the test passes, click Enable SAML Login in New Relic.

    Note: All accounts in New Relic need to be activated by the user by clicking on the Confirm Access button in the activation emails received by users, as shown in the example email below. Until this is done, you will receive an error message when you click Enable SAML Login.

  10. Done!
