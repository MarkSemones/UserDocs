How to Configure SAML 2.0 for TELUS Business Connect

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. Generate the following IDP Metadata and save to a file named metadata.xml:

     
  2. Send an email to TELUS Business Connect support and request that they enable SAML 2.0 for your organization.
  3. Attach the metadata.xml file you created in step 1 to the email message.
  4. TELUS Business Connect will process your request and send you back an SP metadata file.
  5. Open the SP metadata file in a text editor:

  6. In Okta, select the General tab for the TELUS Business Connect app, then click Edit.

    • Copy the value of the entityID field in the SP metadata file into the Audience Restriction field in Okta.

    • Copy the value of the Location field in the SP metadata file into the ACS Url field in Okta.

    • Click Save.

  7. In Okta, select the Sign On tab for the TELUS Business Connect app, then click Edit

    • For Default Relay State enter: https://voicemanager.businessconnect.telus.com/mobile/ssoLogin.

    • Click Save.

  8. Done!

    Notes:

    IdP-initiated flows and SP-initiated flows are supported.

    Just In Time (JIT) provisioning is not supported.

For an SP-initiated flow:

  1. Go to https://voicemanager.businessconnect.telus.com/login/startupSSOLogin.html.

  2. Enter your email address.
  3. Click Continue.
