How to Configure SAML 2.0 for Pathgather

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

Before you begin

Read this before you enable SAML

The Pathgather application is pre-configured to pass a required custom attributes 'Job Title' and 'Department' (which are mapped to title and department in Okta User Base Profile) as part of the SAML assertion - these attribute will be required to use SAML and perform Just in time provisioning for Pathgather. In order to be able to send custom attributes, your Okta organization needs to have the following feature flags turned on to enable the application to work as expected:

  • CLOUD_DIRECTORY (Universal Directory)
  • OKTA_EL_SAML_ATTRIBUTE_UPGRADE
    • With the above feature flags turned on, please ensure that the 'title' and 'department' attributes are populated in the Okta User Base Profile, and have the correct attribute mapping set with AD/LDAP. If this value is empty, Okta will pass a null values to Pathgather.

    2. Send an email to Pathgather support and request that they enable SAML 2.0 for your organization.
    3. Attach the metadata.xml file you created in step 1 to the email message.
    4. In the email, provide them with the following list of attribute keys: 
       first_name, last_name, email, job_title, department
    5. Pathgather will process your request and enable SAML for your account.

    6. Done!

      Notes:

      IdP-initiated flows and SP-initiated flows are supported. For SP initiated flow, go to your Pathgather URL - For example: https://subdomain.pathgather.com

      Just In Time (JIT) provisioning is supported.
