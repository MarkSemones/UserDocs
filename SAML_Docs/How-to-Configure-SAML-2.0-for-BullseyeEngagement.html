How to Configure SAML 2.0 for BullseyeEngagement

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. Log in to BullseyeEngagement as as Administrator.

  2. Navigate to Settings > Integration, then click the Okta logo:

  3. Enter the following:

    • Http Endpoint: Enter https://www.okta.com.

    • x.509 Certificate: Copy and paste the following:

      Sign into the Okta Admin Dashboard to generate this variable.

    • Click Save.

  4. Navigate to Settings > API.

  5. Enter the following:

    • An Application Name, for example, Okta.

    • Click Create.

  6. Make a copy of your Access Token and Application Key:

  7. In Okta, select the General tab for the BullseyeEngagement app, then click Edit.

    • Enter the values you just copied for Access Token and Application Key into the corresponding fields.

    • Click Save.

  8. Done!

    Notes:

    IdP-initiated flows are supported.

    SP-initiated flows and Just In Time (JIT) provisioning are not supported.
