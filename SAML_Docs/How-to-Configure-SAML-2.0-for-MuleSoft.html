How to Configure SAML 2.0 for MuleSoft

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. Login to your MuleSoft - Anypoint Platform account as an administrator.

  2. Click the dropdown icon, then select Access Management.

  3. Click External Identity.

  4. Click the If you would like to configure single sign on with a SAML 2.0 provider you can get started here link, then enter the following (see screen shot at the end of step for reference):

    • Issuer: Copy and paste the following:

      Sign into the Okta Admin Dashboard to generate this variable.

    • Public Key: Copy and paste the following:

      Sign into the Okta Admin Dashboard to generate this variable.

    • Audience:

      • First, generate the following:

        Sign into the Okta Admin Dashboard to generate this variable.

      • Copy the last part of the above generated URL, for example, the text in red in the following example:

        http://www.myCompany.com/exk18o312mrXCbUs21d8

      • Append .anypoint.mulesoft.com, as shown in the following example, and enter it into the Audience field.

        exk18o312mrXCbUs21d8.anypoint.mulesoft.com

    • Sign On URL: Copy and paste the following:

      Sign into the Okta Admin Dashboard to generate this variable.

    • Click Save.

  5. Done!

    Notes:

    IdP-initiated flows and Just In Time (JIT) Provisioning are supported.

    SP-initiated flows are not supported.
