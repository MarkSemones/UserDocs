This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. Log in to ScreenSteps with your credentials and select Admin.

  2. Select Account Management>Remote Authentication>Create Authentication Endpoint.

  3. Add any desired Title.

  4. Copy and paste the following line into the Remote Login URL field: 
    Sign into the Okta Admin dashboard to generate this value.

  5. Okta does not Support Log Out URL, so leave that field blank.

  6. Download the Token Signing X.509 Certificate below. Click Choose a file to upload it.

    Sign into the Okta Admin dashboard to generate this value.

  7. Click Update.

  8. Please use the numeric value at the end of the SAML Consumer URL. Use it as the Authentication Endpoint ID when configuring the app in Okta. (69 in the above example.)
  9. Note: For SP initiated flow, use the SAML Test URL provided on this page. 
  10. Done!
