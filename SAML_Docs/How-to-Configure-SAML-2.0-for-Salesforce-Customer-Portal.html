How to Configure SAML 2.0 for Salesforce Customer Portal

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. SAML for the Customer Portal is dependent on SAML being correctly configured for the Salesforce organization that contains the portal

  2. First setup Salesforce SAML in Okta with your main Salesforce instance and note the Issuer ID

  3. Use that same Issuer ID in the configuration for the Customer Portal.
