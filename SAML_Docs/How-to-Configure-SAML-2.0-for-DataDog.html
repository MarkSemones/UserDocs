How to Configure SAML 2.0 for Datadog

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. Copy and save the following IdP Metadata show below with the filename metadata.xml.

    Sign into the Okta Admin dashboard to generate this value.

  2. Log in to Datadog at https://app.datadoghq.com/account/login with your administrative credentials.

  3. Select the Datadog icon in the bottom right, and then select Configure SAML in the menu that appears, as shown below.

  4. Select Enable SAML, as shown below.

  5. In the Identity Provider Metadata section, select Browse… and navigate to the metadata.xml file you saved in step 1, as shown below. When the file name appears, select Upload File.

  6. Note the Single Sign On Login URL to use for authentication that is shown in the Status section.

  7. If the IdP Initiated Login feature has been activated (optional):

    1. Open your Datadog SAML Configuration page.

    2. Copy your Company ID prefix from the Assertion Consumer Service URL as shown below:

      For example, if your ACS URL is https://app.datadoghq.com/account/saml/assertion/id/[yourCompanyId], you need to copy the /id/[yourCompanyId] value.

    3. In Okta, select the General tab for the DataDog app, then click Edit.

    4. Enter your Company ID as shown below:

    5. Click Save.

  8. Done!

    Notes:

    IdP Initiated Login and Just in Time Provisioning (JIT Provisioning) features are available on request. If you would like access to them, contact Datadog support and describe your scenario. The feature will be available on your account after some configurations on Datadog side. Please refer to http://docs.datadoghq.com/guides/saml/ doc.

    IdP-initiated flows, SP-initiated flows, and Just In Time Provisioning (JIT) are all supported.

    For the SP-initiated flow: Open your Single Sign On Login URL (see step 6).
