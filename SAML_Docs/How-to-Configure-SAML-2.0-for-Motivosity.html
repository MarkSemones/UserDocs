How to Configure SAML 2.0 for Motivosity

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

Before you begin

Read this before you enable SAML

The Motivosity application is pre-configured to pass the custom attributes as part of the SAML assertion. These attributes are required to perform Just in Time (JIT) provisioning for Motivosity. In order to be able to send custom attributes your Okta organization needs to have the following feature flags turned on:


  • CLOUD_DIRECTORY (Universal Directory)
  • OKTA_EL_SAML_ATTRIBUTE_UPGRADE

    • Once these feature flags are turned on, make sure that the attributes are populated in the Okta User Base Profile, and also have the correct attribute mappings set with AD/LDAP (if needed). If this value is empty, Okta will pass a null values to Motivosity.

    The following attributes are supported:


    Display Name Variable Name Data Type
    MV Birth Date mvBirthDate datetime
    MV Hire Date mvHireDate datetime
    MV Is Admin mvIsAdmin boolean
    MV Inventory Manager mvIsInventoryManager boolean
    MV Badge Manager mvIsBadgeManager boolean
    MV Payroll ID mvPayrollId string
    MV Manager Email mvManagerEmail string

    Standard Universal Directory user fields:


  • firstName
  • lastName
  • email
  • title
  • nickName
  • primaryPhone
  • department
