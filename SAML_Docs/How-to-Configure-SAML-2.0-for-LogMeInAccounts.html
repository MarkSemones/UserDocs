How to Configure SAML 2.0 for LogMeIn Accounts

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

Before you Begin

Read this before you enable SAML

SAML 2.0 is supported by Common Login Service (CLS), so you must configure the LogMeIn Accounts application in Okta to support SAML for all CLS-enabled products (https://accounts.logme.in/, https://secure.logmein.com/, https://www.join.me/, https://www.meldium.com/, https://www.cubby.com/, https://www.appguru.com/).

  1. Send an email to LogMeIn support and request that they enable SAML 2.0 for your account.

  2. Attach the following to your request:

    • IDP metadata file: Copy and paste the following:

      Sign in to Okta Admin app to have this variable generated for you.

    • Certificate: Download, then upload and attach the following:

      Sign into the Okta Admin Dashboard to generate this variable.

    • SignOn URL: Copy and paste the following:

      Sign into the Okta Admin Dashboard to generate this variable.

    • Domain: [YourDomain]

  3. Done!


    Notes:

    IdP-initiated flows, SP-initiated flows, and Just In Time (JIT) provisioning are all supported.


    SP-initiated Flow:

    SP-initiated flow is available for all CLS-enabled products (https://accounts.logme.in/, https://secure.logmein.com/, https://www.join.me/, https://www.meldium.com/, https://www.cubby.com/, https://www.appguru.com/).

    For the SP-initiated flow you should open login page for the CLS-enabled product, enter your email, then click Log in button.


    IDP-initiated Flow:

    IDP-initiated flow is available only for the LogMeIn Accounts, LogMeIn SAML, Join.Me SAML apps.

    To initiate IDP flow for the LogMeIn SAML or Join.Me SAML apps you need to add these apps in OKTA (without any additional configuration).
