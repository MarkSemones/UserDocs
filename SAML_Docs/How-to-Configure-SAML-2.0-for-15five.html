How to Configure SAML 2.0 for 15Five

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

Before You Begin

Read this before you enable SAML

The 15Five application is pre-configured to pass the custom attribute managerEmail as part of the SAML assertion. In order to be able to send the custom attribute, your Okta organization needs to have the following feature flags turned on:


  • CLOUD_DIRECTORY (Universal Directory)

  • OKTA_EL_SAML_ATTRIBUTE_UPGRADE


    • Once these feature flags are turned on, make sure that the custom attribute managerEmail is populated in the Okta User Base Profile (see instructions below), and also has the correct attribute mappings set with AD/LDAP (if needed). If this value is empty, Okta will pass a null values to 15Five.

    Adding a Custom Attribute

    Follow these instructions to add the manageEmail custom attribute:

    1. In Okta, from the Admin dashboard, navigate to Directory > Profile Editor.

    2. Select Okta from Filters.

    3. Click the edit Profile button:

    4. Click the Add Attribute button:

    5. Enter the following:

      • Display name: managerEmail.

      • Variable name: managerEmail.

      • Click Add Attribute.
