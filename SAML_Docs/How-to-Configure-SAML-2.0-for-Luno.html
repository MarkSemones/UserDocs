How to Configure SAML 2.0 for Luno

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

Before you begin

Read this before you enable SAML

The Luno application is pre-configured to pass the required custom attribute EmployeeID (which is mapped to employeeNumber in Okta User Base Profile) as part of the SAML assertion. This attribute is required to use SAML and perform Just in Time (JIT) provisioning for Luno. In order to be able to send custom attributes your Okta organization needs to have the following feature flags turned on:


  • CLOUD_DIRECTORY (Universal Directory)

  • OKTA_EL_SAML_ATTRIBUTE_UPGRADE


    • Once these feature flags are turned on, make sure that the employeeNumber attribute is populated in the Okta User Base Profile, and also has the correct attribute mapping set with AD/LDAP. If this value is empty, Okta will pass a null value to Luno.

    1. Send an email to Luno Support and request that they enable SAML for your organization.

    2. Copy, then save the following IDP Metadata file as metadata.xml and attach it to the email:

      Sign in to Okta Admin app to have this variable generated for you.

    3. Luno will process your request. Once you receive an email confirming that SAML has been enabled, you can start assigning users to the application.

    4. Done!

      Notes:

      IdP-initiated flows and SP-initiated flows are supported.

      Just In Time (JIT) provisioning is not supported.
