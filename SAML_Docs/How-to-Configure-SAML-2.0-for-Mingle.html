How to Configure SAML 2.0 for Mingle

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. In Okta, under the General tab, add your subdomain to the Sub Domain field, shown below. For example, if your Mingle site is https://acme.mingle.thoughtworks.com, enter acme as your sub domain.

    Hint: Access the Sub domain field in Okta by selecting the General tab in the Mingle application, as shown below.

  2. In Okta, on the Sign On tab, download the IdP metadata file by selecting the Identity Provider metadata link, as shown below.

  3. Log on to Mingle and select Admin > SSO Config, as shown below.

  4. Select Choose File in Mingle, as shown below. Upload the file containing the IdP Metadata you downloaded in step 2. The metadata displays after it is uploaded.

  5. Click Save.

     

  6.  Done!
