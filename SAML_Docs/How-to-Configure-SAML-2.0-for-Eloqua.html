How to Configure SAML 2.0 for Eloqua

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. Sign in tho Eloqua. Select Setup > Management > User Management > Single Sign-On.

  2. Then, navigate to Setup > Users > Single Sign-On.

  3. Click Single Sign-On and select Identity Provider Settings on the submenu that displays.

  4. Click the Upload Identity Provider from Metadata button, as shown below.

  5. Enter Okta for the name of the identity provider.

  6. Save the following metadata as an xml file. Click the Browse button and navigate to the location of this file.

    Sign in to the Okta Admin app to have this variable generated for you.

  7. Click the Save button.

  8. Click Edit next to the identity provider you just created.

  9. Select Assertion contains the Email Address from the User object under User Identity Mapping, as shown below.

  10. Copy the Service Provider Entity field in the identity provider you just created. In Okta, select the General tab in the Eloqua application and add the information to the Service Provider Entity field.

  11. Done!
