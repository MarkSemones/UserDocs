How to Configure SAML 2.0 for Cisco WebEx

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. Contact WebEx Support to turn on SAML for your WebEx account. Their support team will notify you once SAML has been enabled.

  2. Once SAML is enabled, log in to your WebEx instance as an Admin. The admin page can be accessed from https://SITENAME.webex.com/admin.php, replacing SITENAME with the actual name of your WebEx branded site.

  3. Click the SSO Configuration tab on the left menu.
  4. Click Site Certificate Manager. Save the following certificate in PEM format as cert.pem, and upload it in WebEx. 
    Sign into the Okta Admin Dashboard to generate this variable.
  5. Click Federated Web SSO Configuration to display the Federated Web SSO Configuration dialog box.

  6. Enter the following configuration information:

    • Federation Protocol: Select SAML 2.0 from the drop down menu.

      The fields displayed in the Federated Web SSO Configuration dialog box vary based on the selected federation protocol. By default, the configuration fields for SAML 2.0 will be displayed each time the Federated Web SSO Configuration dialog box is opened.

    • Issuer for SAML (IdP ID): Copy and paste the following in this field: Sign into the Okta Admin Dashboard to generate this variable.
    • Customer SSO Service Login URL: Copy and paste the following in this field: Sign into the Okta Admin Dashboard to generate this variable.
    • NameID Format: Select Unspecified
    • AuthnContextClassRef: Make sure this is set to the following: urn:oasis:names:tc:SAML:2.0:ac:classes:PasswordProtectedTransport

  7. Click Update
  8. Done!
