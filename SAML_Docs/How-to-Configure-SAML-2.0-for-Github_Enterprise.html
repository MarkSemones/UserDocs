How to Configure SAML 2.0 for Github Enterprise

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. Sign in to your Github Admin Console at https://[hostName]/setup where [hostName] is your Github Enterprise host name.

  2. Select Authentication on the left menu bar. Next, select SAML, as shown below.

  3. Copy following URL in to the Sign On URL field:

    Sign into the Okta Admin dashboard to generate this value.

  4. Select the link below to download a Verification Certificate.

    Sign in to the Okta Admin dashboard to generate this value.

  5. After the certificated is downloaded, select the Browse link and upload it.

  6. Scroll down and select Save.

  7. In Okta, on the General tab for the Github Enterprise app, enter your GitHub Enterprise Domain URL/Hostname. This value is on the Authentication settings page that you just saved in your GitHub Admin Console.

  8. Done!

    Note: IdP-initiated and SP-initiated flows are supported. Just In Time (JIT) provisioning is not supported.
