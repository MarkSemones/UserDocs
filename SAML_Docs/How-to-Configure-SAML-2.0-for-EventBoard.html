How to Configure SAML 2.0 for EventBoard

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. Send an email message to EventBoard support and request that they enable SAML for your organization.

  2. Copy and paste the following Entity ID into the email message.

    Sign into the Okta Admin app to generate this variable.

  3. Copy and paste the following Sign-in URL into the email message.

    Sign into the Okta Admin app to generate this variable.

  4. Select the link below to download an x509 Certificate. Attach the downloaded certificate to the email message.

    Sign in to the Okta Admin app to generate this link.

  5. EventBoard will process your request. When you receive a confirmation that SAML is enabled, you can start assigning users to the application.

  6. Done!

    Note: Only SP-initiated flows are supported. IdP-initiated flows and and Just In Time (JIT) provisioning are not supported.

    EventBoard only supports an SP-initiated flow. For the convenience of your end users, Okta recommends that you hide the EventBoard app and set up a Bookmark app with the EventBoard logo. For instructions on setting up a Bookmark app and hiding the original app, see Simulating an IDP-initiated Flow with the Bookmark App.

    The link for an SP-initiated flow is: https://[yourSubDomain].eventboard.io/login/, where [yourSubDomain] is the domain you have already set up.
