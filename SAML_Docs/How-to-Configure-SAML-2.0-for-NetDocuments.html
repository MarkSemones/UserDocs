How to Configure SAML 2.0 for NetDocuments

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. Sign into NetDocuments.

  2. Select Admin in the upper right corner.

  3. Navigate to Add and remove users and groups.
  4. Select Configure advanced authentication requirements at the bottom of the page, as shown below.

  5. In the Federated Identity section, select Okta as the Federated identity server type, as shown below.

  6. In Okta, copy the link associated with the text Identity provider metadata, shown below.



    IMPORTANT: Only copy the link to the metadata. Do not copy and paste the metadata.
  7. Paste that link into the Federated Metadata document URL field in Step 3, as shown below.



  8. Note the URL to login to NetDocuments that is provided, as shown above.
  9. Done!
