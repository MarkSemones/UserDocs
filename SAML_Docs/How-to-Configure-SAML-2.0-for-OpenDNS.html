How to Configure SAML 2.0 for OpenDNS

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. Generate the following IDP Metadata and save it to a file named metadata.xml:

     
  2. Log into OpenDNS as an administrator.
  3. Select the Configuration tab.
  4. Select System Settings > Login Security on the left navigational panel.
  5. Select Okta.
  6. Click next >>.
  7. The Instructions page opens, click next >>.
  8. The Upload Metadata page opens, select Browse, then select the metadata.xml file you saved in step 1.
  9. The Validate page opens, select Test your SAML configuration.
  10. Enter your OpenDNS email into the corresponding field, then click Login via SAML.
  11. Enter your Okta Username and Password, then click Sign In.
  12. Wait until a Your SAML settings have been properly configured! message appears, then click Save Configuration and notify users.
  13. Done!

    Notes:

    SP-initiated flows and IDP-initiated flows are supported.

    Just in Time (JIT) provisioning is not supported.

    For SP-initiated flows, go to https://login.opendns.com/sso/okta. Enter your OpenDNS email, then click SIGN IN.
