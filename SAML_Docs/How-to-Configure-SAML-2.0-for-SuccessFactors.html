How to Configure SAML 2.0 for SuccessFactors

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

Read this before you enable SAML

Enabling SAML will affect all users who use this application, which means that users will not be able to sign-in through their regular log-in page. They will only be able to access the app through the Okta service.

Contact SuccessFactors Customer Support

Contact SuccessFactors' Customer Support and ask them to enable SAML 2.0 Single Sign-On for your account. The following information is needed:

  1. SAML Issuer:

    Sign in to the Okta Admin app to have this variable generated for you.

  2. Relying Party Description: localhost

  3. Require Assertion Signature: Yes

  4. Enable SAML Flag: Enabled

  5. Login Request Signature: No

  6. Login Response Signature: Yes

  7. SAML Profile: Browser/Post Profile

  8. Enforce Certificate Valid Period: Yes

  9. SAML Verifying Certificate:

    Sign in to the Okta Admin app to have this variable generated for you
