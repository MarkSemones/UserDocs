How to Configure SAML 2.0 for Instructure Bridge

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. Send an email to Instructure Bridge support and request that they enable SAML 2.0 for your account.

  2. Copy and attach the following:

    • SAML IdP Entity ID:

      Sign into the Okta Admin Dashboard to generate this variable.

    • SAML Certificate Fingerprint:

      Sign into the Okta Admin Dashboard to generate this variable.

    • SAML Identifier Format:

      urn:oasis:names:tc:SAML:2.0:nameid-format:persistent

    • SAML Authentication Context:

      urn:oasis:names:tc:SAML:2.0:ac:classes:PasswordProtectedTransport

  3. Instructure Bridge support will process your request. After receiving a confirmation email, you can start assigning people to the application.

  4. Done!

    Notes:

    IdP-initiated flows, SP-initiated flows, and Just In Time (JIT) provisioning are all supported.

    For SP-initiated flows:

    Go to https://[yourSubDomain].bridgeapp.com/.
