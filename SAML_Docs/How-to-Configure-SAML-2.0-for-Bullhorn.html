How to Configure SAML 2.0 for Bullhorn

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

 

  1. Email the Bullhorn support team and request that they enable SAML.
  2. Attach the following data to the email:
    •   Identity Provider Issuer (Entity ID):Sign into the Okta Admin Dashboard to generate this variable.
    • Certificate:Sign into the Okta Admin Dashboard to generate this variable.
    • Identity Provider URL:Sign into the Okta Admin Dashboard to generate this variable.
  3. Bullhorn will process your request and after receiving a confirmation email from them, you can start assigning people to the application.

  4. Done!

    Notes:

    IDP-initiated flows are supported.

    SP-initiated flows and Just In Time (JIT) provisioning are not supported.
