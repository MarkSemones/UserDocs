How to Configure SAML 2.0 for OfficeSpace Software

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. Login to OfficeSpace Software as an administrator.

  2. Select the Connectors tab:

  3. Enter the following values into the corresponding fields (see screen shot at end of step for reference):

    • Enabled: true

    • Is primary provider: true
    • Logout provider url: Copy and paste the following into this field: Sign into the Okta Admin Dashboard to generate this variable.
    • Client idp target url: Copy and paste the following into this field: Sign into the Okta Admin dashboard to generate this value.
    • Client idp certificate fingerprint: Copy and paste the following into this field: Sign into the Okta Admin dashboard to generate this value.
  4. Click Save

  5. In the Okta Dashboard for the OfficeSpace Software application, select Okta username for the Default username format, as shown below. 

  6. Done!

    Note: IdP-initiated flows, SP-initiated flows, and Just in Time (JIT) provisioning are all supported.
