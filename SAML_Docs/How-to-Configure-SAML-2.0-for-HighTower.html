How to Configure SAML 2.0 for Hightower

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. Contact Hightower Support and request that they enable SAML for your app.

  2. Provide the following information:

    • x.509 Certificate. Either:

      • Download then attach the x.509 Certificate in .cert format:

        Sign into the Okta Admin Dashboard to generate this variable.

      • Cut and paste the x.509 Certificate in text format:

        Sign into the Okta Admin Dashboard to generate this variable.

    • Login URL/SignOn URL:

      Sign into the Okta Admin Dashboard to generate this variable.

    • IDP Issuer/Entity ID:

      Sign into the Okta Admin Dashboard to generate this variable.

  3. Hightower will process your request and provide you with the ACS URL and the metadata (Audience Restriction in Okta).

  4. In Okta, select the General for the Hightower app, and enter the ACS URL and the Audience Restriction you received from Hightower, as shown below:

  5. Click Save.

  6. Done!

    Notes:

    SP-initiated flows and Just In Time (JIT) provisioning are supported.

    IdP-initiated flows are not supported.


    For SP-initiated Flows:

    Go to https://app.gethightower.com/users/sign_in.
