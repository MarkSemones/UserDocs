How to Configure SAML 2.0 for Xendo

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. Email Xendo support and request that they enable SAML 2.0 Single Sign On for your account.

  2. Xendo support will send you a link to a secure page where you can upload settings. Navigate to that page and upload the following three items.

    1. Copy and paste the following X.509 Certificate. Be sure to copy the Begin Certificate and End Certificate lines.

      Sign into the Okta Admin dashboard to generate this value.

    2. Copy and paste the following Sign On URL.

      Sign into the Okta Admin dashboard to generate this value.

    3. Copy and paste the following IDP Issuer ID.

      Sign into the Okta Admin dashboard to generate this value.
  3. After you enter these values, Xendo will configure SAML 2.0 for your organization and notify you when you can test the integration with the provided single sign-on URL.
  4. When the customer test is completed successfully, assign the Xendo app to users in Okta.

  5. Done!

    Note: IdP-initiated flows and Just In Time (JIT) provisioning are supported., SP-initiated flows are not supported.
