How to Configure SAML 2.0 for Engagedly

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. Log in to Engagedly as a Site Administrator.

  2. Navigate to Settings, then select SAML SSO.

  3. Make a note of the following (in the Setting for Identity Provider section):

    • ACS URL.

    • Entity ID.

  4. Enter the following (in the Setting from Identity Provider section):

    • Check the Enable SAML box.

    • Identity Provider Issuer: Copy and paste the following:

      Sign into the Okta Admin Dashboard to generate this variable.

    • Identity Provider Single Sign-On URL: Copy and paste the following:

      Sign into the Okta Admin Dashboard to generate this variable.

    • Identity Provider X.509 Certificate: Copy and paste the following (PEM text format):

      Sign into the Okta Admin Dashboard to generate this variable.

    • Click Save.

  5. In Okta, select the General tab for the Engagedly app, then click Edit and enter the following:

    • ACS URL: Enter the value you saved earlier.

    • Entity ID: Enter the value you saved earlier.

    • Click Save.

  6. Done!

    Notes:

    IdP-initiated flows and SP-initiated flows are both supported.

    Just In Time (JIT) Provisioning is not supported.
