How to Configure SAML 2.0 for Cisco Meraki Dashboard

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. Sign in to Cisco Meraki Dashboard as an administrator.

  2. Navigate to Organization > Settings > SAML Configuration. The screen shown below opens.

  3. For the SAML SSO setting, select SAML SSO enabled from the dropdown list, as shown above.

  4. Enter the X.509 cert SHA1 fingerprint, as shown above.

    Use the fingerprint below and convert it to the format Cisco Meraki Dashboard requires. Change all letters in the fingerprint below to upper case and insert colons after every second character. For example, if the fingerprint below were 1x2y3z, change it to 1X:2Y:3Z using a text editor, before pasting it into the form. 

    Sign in to the Okta Admin app to generate this variable.

  5. Enter the following SLO logout URL (optional), as shown above.

    Sign into the Okta Admin dashboard to generate this value.

  6. Select Save Changes.

  7. Navigate to Organization > Administrator > SAML administrator roles. 
  8. Select Add SAML role and add a role with a name and the necessary privileges.

  9. Done!

Note: An IdP-initiated flow and Just In Time (JIT) provisioning are supported. An SP-initiated flow is not supported.
