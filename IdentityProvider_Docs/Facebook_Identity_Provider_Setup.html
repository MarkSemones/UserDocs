Configure Facebook as an Identity Provider

  1. Go to developers.facebook.com and register for a developer account if you haven't already done so.

  2. Choose My Apps from the toolbar.
  3. Choose Add a New App from the dropdown.
  4. Choose Website from the Add a New App window.
  5. Choose an existing app or enter a name for a new app. Users see the app name when they authenticate to your app.
  6. If you entered a new name, select Create New Facebook App ID and enter the requested information. If you're not sure which category to select, choose Utilities.
  7. Choose the blue Create App ID button to save the configuration.
  8. In the Quick Start for Website page, choose Skip Quick Start. Facebook displays your app's dashboard.
  9. Save the App ID and App Secret values so you can add them to the Okta configuration in the next section.

Configure Facebook as an Identity Provider in Okta

  1. Log in to your Okta org and choose Security > Identity Providers.
  2. Choose Add Identity Provider.
  3. Choose Configure, then Configure Identity Provider.
  4. Enter Facebook's App ID in the Okta Client Id field.
  5. Enter Facebook's App Secret in the Okta Client Secret field.
  6. Save the Okta Facebook configuration.
  7. From the Identity Providers page, find the Facebook listing and copy the value in the Redirect URI field, so you can add it to the Facebook configuration in the next section.

Return to Facebook

  1. In your Facebook app configuration, select Settings in the left pane.
  2. Choose Add Platform.
  3. Choose Website.
  4. In the Site URL field, enter the Okta redirect URI that you copied.
  5. Click Save Changes.

Facebook Scopes

The public_profile and email OAuth scopes are automatically included by Okta. If your app requires more than public_profile, email, request a Login Review.

For more information on Facebook scopes, see Permissions.
