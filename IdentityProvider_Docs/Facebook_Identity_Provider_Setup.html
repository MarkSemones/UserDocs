Identity Provider Set Up for Facebook

  1. Go to developers.facebook.com and register for a developer account. You may be prompted to register if you have not previously used the Facebook Developer site.

  2. Click My Apps from the toolbar and select Add a New App from the dropdown.
  3. Select Website from the Add a New App window.
  4. Choose a name of your app. This name is what your users will see when they authenticate to your app.
  5. Click Create New Facebook App ID.
  6. You will need to choose a category for your app in the Create a New App ID window. If you are unsure what to select, we suggest selecting the Utilities category. Click the blue Create App ID button when you are done.
  7. When you see Quick Start for Website, click Skip Quick Start. You should now see the Dashboard for the app you are creating.

Finish configuring the Facebook Identity Provider in Okta

  1. Return to the browser tab where you were configuring Identity Providers in Okta.
  2. Use the App ID from above as the Okta Client Id.
  3. Use the App Secret from above as the Okta Client Secret.

Return to Facebook

  1. In your Facebook app configuration, click Settings in the left pane.
  2. Click Add Platform.
  3. Select Website.
  4. In the Site URL field in the "Website" section, enter the Redirect URI you obtained above.
  5. Click Save Changes.

Note regarding Facebook scopes

The public_profile and email OAuth scopes are automatically included by Okta. If your app requires more than public_profile, email, it will require a Login Review. For more information on Facebook scopes, go to the Permissions section on their developer site.
